Some desktop applications require an X.509 certificate--a special packet of data that represents your identity--to be loaded into them so that they can connect to secured services. Similarly, some web applications require such a certificate to be loaded into your browser. This page can provide you with such a certificate. If you are not sure what this is, then you probably don't need one.

This service lets you create and download a USVAO X.509 certificate plus private key suitable for loading into applications that run directly on your machine—such as a web browser. This will be a so-called "end-entity" certificate (i.e. not a proxy). It can be delivered along with its associated private key in either PKCS12 or PEM format.


Lifetime:    The time before the certificate expires
Package Key:       A password for sealing the credential file that will be downloaded. You will need this when you load the certificate into your application.
Format:    Preferred by browsers and Java.
   Commonly used by grid computing tools.
Account Password:       The password to your account
