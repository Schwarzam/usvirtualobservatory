VAO Logins Site Not Vulnerable to the Heartbleed Bug

April 10, 2014

This week press releases announced the discovery of a serious security vulnerability, dubbed the Heartbleed bug, in the OpenSSL software. OpenSSL is a library used by many web sites, including this one, to enable secure, encrypted access to web pages using HTTPS. More information about this bug can be found at heartbleed.com. One critical fact for this site is that this bug was introduced in version 1.0.1 of the OpenSSL software.

It turns out that the VAO Logins site has been using OpenSSL version 1.0.0j; consequently, we are and have not been vulnerable to this bug. Tests using Heartbleed detection tools confirm this. We will soon be releasing a new version of the site using the latest, non-vulnerable OpenSSL library (v1.0.1g or later).

Fortunately, it is not necessary for us to revoke or reissue certificates. Even better, users should not need to change the their passwords. An exception would be for users whose VAO Login passwords are the same as used on other affected sites (generally frowned upon). You can change your password at any time via our password page.
