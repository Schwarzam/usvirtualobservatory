A. General Questions about VAO Logons

A.1. What is a VAO Logon? What is an "identity"?

Also called your VAO Identity, it is a user name and password that you can use to log into web portals in the VO.

Your identity also includes all your profile information such as your full name, institution, email address, etc. The VAO Login Portal provides you with tools for managing your identity.

A.2. Why create a VAO Logon?

Because your VAO logon is supported at a number of VO portals, you will now have fewer usernames and passwords to keep track of.

Your VAO login name gives you added power while working in the VO through its single sign-on capabilities--once you're logged into the VAO, you can move freely between portals without retyping you password. More importantly, portals and services can pass private data back and forth securely with each other on your behalf.

A.3. How do I get a VAO Logon?

You can create your logon by filling out the VAO registration form. We'll then send you a quick email confirmation which you must respond to so that we know you are contactable and not a robot. Once that is done, you'll set your username and password, and you're done.

A.4. How is this different from (or similar to) other federated login systems?

The VAO Login Services uses the OpenID, a community standard for federated, single sign-on authentication. This same standard is supported by widely across the web (including Google and Facebook). In this way, VAO Logins are similar to the Google and Facebook OpenID Login services.

The most important way the VAO Login Services is different from other OpenID services is it provides special support for interoperability between VO services using X.509 certificates (in accordance with standards by the International Virtual Observatory Alliance, or IVOA).

A.5. What is OpenID? Do you support it?

OpenID is a community standard for federated, single sign-on authentication through the web. Yes, we support it (but more on that below).

Widely supported across the web, OpenID allows a user to log into many web sites with a single username and password. When one logs into a OpenID-compatible site (i.e. a portal), the user's browser is sent to the OpenID provider's web site to enter a password. Thus, a user's password is never shared with the portal. The VAO Login Services serve as an OpenID provider.

OpenID also supports a secure way to share user information (like full name and email address) between the provider and the portal in a way that respects the user's privacy. In particular, before the VAO service will share any information with a portal, it asks for the user's permission.

The VAO Login Services complies with the standard for OpenID providers; however, typical use may appear a little different from other providers. An OpenID identity is usually given in the form of a URL. While the VAO Login Services supports this form, VAO users typically do not see or enter this URL to login; instead, they use a short, more traditional username.

A.6. Can I use my OpenID with non-VO portals and web sites?

Often, yes, but it depends on the web site. That is, many sites will accept a login from any OpenID provider. In this case, the user typically enters their OpenID URL instead of a username. (See B.5 for information about your VAO OpenID URL.) However, some web site administrators may choose to only accept logins from certain OpenID providers.

A.7. Can I use my XXX (e.g. Google) OpenID to log into VO portals?

In principle, yes, but typically no; it depends on the particular VO portal. To take advantage of some of the VO portal interoperability features, the user needs to log in using a VAO login. However, an administrator for a VAO-compatible portal may choose to accept OpenIDs from any OpenID provider; in this case, a Google OpenID can be used. Check the portal's documentation for details.

A.8. What is OAuth? Do you support it?

OAuth is another commmunity standard for security on the web. It is complementary to OpenID in that it enables authorization--that is, permissions--across portals and services on the web. That is, it provides a mechanism for giving one portal permission to access something that belongs to a user on another portal. Permission is granted to one portal in the form of a secure token that can be presented to the second portal.

Currently, the VAO Login Services does not directly issue or use OAuth tokens at this time; although, we plan to add use soon. Note, however, that there are VAO-compatible portals that do issue and use OAuth tokens with VAO Logins providing the authentication required.

A.9. What's an X.509 certificate? Do I need one?

An X.509 certificate is a standard file that securely represent your identity in a electronically verifiable way. You do not need one to use VAO Login Services.

An X.509 certificate is an example of a so-called public key certificate. It provides a way for one program to prove the identity of its user without the user having to type in a password.

The VAO Login Services and compatible portals may use X.509 certificates behind the scenes if necessary to enable interoperability between services across the VO. For example, a portal may allow you to access all of your proprietary data in the VO through a single tool (e.g. a vizualizer); to obtain access, the portal will use an X.509 certificate representing your identity to access secure data access services on your behalf. In this application, you do not need to handle the certificates yourself.

Some desktop or grid-based tools require that you install an X.509 certificate onto your local computer, into the tool, or into your web browser. If you need a certificate for this purpose, you can obtain one through our Certificate Download Page. Most VAO users, however, will not need this.

A.10. Do you have a privacy policy? Terms and Conditions Agreement?

Yes. You can view the VOA Login Services' privacy policy and the conditions for use here.

B. Using and Managing your VAO Logon

B.1. How do I use my VAO Logon to login somewhere?

Typically, a VAO-compatible portal will indicate that they support VAO Logins, say, on their home page and provide a "Login" link. If you are not already logged in the VAO, then clicking on this link will send you to the login page at sso.usvao.org. Here you can enter your username and password. If you do this successfully, you may then be asked to confirm that it is okay to share certain information with the portal (if they've asked for any). You are next returned back to the portal website, logged in and ready to go. If you are already logged into the VAO when you click the "Login" link, it is not necessary to enter your username and password; you will just enter the portal directly.

B.2. Why am I asked to confirm that I want to log in to a portal?

Sometimes the portal will want information about you before it allows you to use it. Usually, this information helps with registering you with that site: it can fill in a registration form with this information so that you don't have to type it in again. Nevertheless, we will not share your information unless you give us permission to do so. The confirmation page requests that permission.

Some portals have registered with us as being VAO-compatible and have agreed to our privacy policy. If we notice that you are logging in from a recognized portal, we'll tell you on the confirmation page; you can feel more confidant about sharing your information from these portals. If we don't recognize the portal, you should pay closer attention who the portal represents and what its purpose is when deciding to share your information. This is especially true if the portal is asking for a secure certificate.

When you give your confirmation, you will have the choice to allow VAO Login Service to remember you choices via a check box. If you check this box, you will not see the confirmation page the next time you visit the site. You can change these choices later via the preferences page.

B.3. Why do you share some of my personal information?

Often, users are required to register with a portal before they can use it. As part of registration, users must provide information about themselves. When he VAO Login Service can provide any of this registration data to the portal upon request, it can save the user having to type it in again.

B.4. What if I move to a new home institution or change my email address?

You can change your person profile information, which includes the name of your home institution, your telephone number, your home country, and your email address, using our Profile Editing Page. By updating your information when you move, the new data can make its way out to all the VO portals you visit the next time you visit them.

B.5. You say VAO Logons uses OpenID. What's my OpenID URL? How do I use my VAO OpenID to log into non-VAO sites?

An OpenID identity is represented as a URL. Normally, VAO users don't make use of this URL directly; instead, they log in using a traditional username (which is shorter and easier to remember than a URL. However, if you want to log into a non-VAO site using OpenID, you would use the URL-form of your VAO identity.

A VAO user's OpenID URL has the following form:

http://sso.usvao.org/openid/id/username
where username is the username you usually log in with. You can use this URL to log into a non-VAO-compatible portal if supports logins from any OpenID provider. Simply enter this URL where prompted for an OpenID.

Note if you control your own web-site, the OpenID standard allows you to define you own OpenID URL. Consult the OpenID standard for details.

B.6. I forgot my username! What can I do?

Visit our Password Management Page to reset your password. As part of the process, we'll send you a secret token by email that you will use to reset your password.

B.7. I forgot my username! What can I do?

Visit our Password Management Page to request an email that will remind you of your username (or usernames).

B.8. Can I have more than one login?

We discourage users from having more than one login. This makes interoperability between archives and portals harder.

Nevertheless, it's not uncommon for a user to unknowingly register with the VAO multiple times. (For example, a user may have forgotten having registered before.) The Username Reminder Tool on the Password Management Page allows you to find all of the usernames associated with any of your email addresses. If you wish to disable any usernames you do not use, do not hesitate to contact us at vaosso at ncsa.illinoi.edu.

B.9. Do you support group accounts? Can I share a login?

VAO Logins does not support group accounts, and sharing a VAO login among multiple users is strongly discouraged. You should not share your password with another user.

VO-compatible archives should instead support authorization groups in which distinct users can be added as members and have access to common collections of data. VAO Login Services itself does not currently support authorization groups, but we may in the future to support data sharing across the VO.

B.10. Should I be worried about phishing through a copy-cat site?

Yes, but there is an easy way to avoid it. The VAO Login page, https://sso.usvao.org/openid/, uses SSL to ensure a secure, encrypted login process. SSL requires that sites verify they are who they say they are with a site certificate that can be check by your browser. If this site verification is successful, your browser will indicate this by URL location bar; here's what Firefox looks like:

Firefox shows a lock icon to indicate that the site is verified. Hovering over the lock with your mouse will show the name of the certificate authority ("Internet2") used to verify the site (see also Question B.11).

Look for this lock icon and confirm that the location is sso.usvao.org to ensure that you are using the genuine VAO site to login.

B.11 What is meant by "verified by Internet2/InCommon" refer to?
The VAO Login page, https://sso.usvao.org/openid/, uses SSL to ensure a secure, encrypted login process and to prevent phishing attacks (see Question B.10). This requires us to provide to browsers a certificate that certifies that we are who we say we are. Our site certificate has been issued by the Internet2 InCommon Certificate Authority (used by many US universities) which is recognized by all major browsers automatically. We note this fact on our login page via the statement:
This page is served from sso.usvao.org, verified by Internet2/InCommon

This statement lets users know that they can confirm that they are using a legitimate login service by looking at their browser's URL location bar. See Question B.10 for more details.

C. For Portal Developers/Maintainers: Supporting VAO Logons

C.1. I run a web site. How do I support VAO Logons?

Consult our documentation for portal developers for details. The VAO Login Services uses OpenID as the mechanism for logging into a portal. Since the VAO service is fully OpenID-compliant, you can use any OpenID library to support VAO logins; our support documentation can advise on VAO-specific features (such as what user attributes we can provide). Note, though, that the VAO also provides a customized toolkit called VAOLogin which is designed to make it easier to integrate support for VAO Logins.

C.2. My web site already uses federated/single sign-on logins, and it's not OpenID. Can I also support VAO Logons?

Yes. Soon, we will publish through our portal developers' support page recommended ways to augment your portal to support VAO Logins. This will allow you to use your own single sign-on system while still support interoperability with other VO portals.

C.3. Must my site support X.509 certificates in order to support VAO Logins?

No. Use of X.509 certificates are optional.

You only need to support certificates if you want your portal to be able to access secured VO services at another VO site. The portal developers' support page describes how to obtain a temporary certificate representing a user as part of a VAO user session. The VAOLogin toolkit can handle this transparently.
