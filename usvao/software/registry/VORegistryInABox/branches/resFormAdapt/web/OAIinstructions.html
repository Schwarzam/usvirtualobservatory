Accessing Repository Resources

Once you have entered resources into your registry and have published them, you can acess
them with the Open Archives Initiative compliant Harvesting Interface included with this package.  It is a Perl script called "oai.pl" and can be found in the directory where the CGI
scripts were installed on your Web server when this package was installed, e.g. the voregistry directory off of your Web Server's cgi directory.

An OAI-compliant harvesstin interface recognizes 6 verbs for accessing various parts of the data.  Here is a synopsis of them, as well as a sample HTTP address to access them.  Here we are using "http://mywebserver.org/cgi-bin/voregistry/oai.pl for the pointer to the harvesting interface; you would substitute your actual web address for you harvesting interface here.

1. Identify: Identify the Registry - This will list general information about the Registry.
No other parameters are required besides the verb.

Example: http://mywebserver.org/cgi-bin/voregistry/oai.pl?
		verb=Identify
2. GetRecord: Return a single record from the registry that meets the criteria specified.  Here, in addition to the verb GetRecord, one must also specify a unique identifier for the resource as well as the metadataPrefix, which describes the format of the output data.  The
current registry knows about two formats: oai_dc which is an OAI Dublin Core format, and vor, which is a VOResource format which contains  more information.
Example: http://mywebserver.org/cgi-bin/voregistry/oai.pl?verb=GetRecord
		&identifier=oai:arXiv.org:cs/0112017
		&metadataPrefix=oai_dc

3. ListRecords: This verb is used to harvest records from a repository. All records within the criteria will be returned. In addition to the verb ListRecords, one must also specify a unique identifier for the resource as well as the metadataPrefix, which describes the format of the output data.  The current registry knows about two formats: oai_dc which is an OAI Dublin Core format, and vor, which is a VOResource format which contains  more information.

Example: http://mywebserver.org/cgi-bin/voregistry/oai.pl?verb=ListRecords
		&metadataPrefix=oai_dc

4. ListIdentifiers: This verb is an abbreviated form of ListRecords, retrieving only headers  rather than records. A metadataPrefix describing the the data output must be included. Optional arguments permit selective harvesting of headers based on set membership and/or datestamp. Current sets for this Publishing Registry are BrowserBasedService, DataCollection, Organization, Project, Resource, and SIAService.

Example: http://mywebserver.org/cgi-bin/voregistry/oai.pl?verb=ListIdentifiers
		&metadataPrefix=oai_dc&set=Project
5. ListMetadataFormats: This verb is used to retrieve the metadata formats available from a repository.  An optional argument restricts the request to the formats available for a specific item.

Example: http://mywebserver.org/cgi-bin/voregistry/oai.pl?verb=ListMetadataFormats

6. ListSets : This verb is used to retrieve the set structure of a repository, useful for selective harvesting.

Example: http://mywebserver.org/cgi-bin/voregistry/oai.pl?verb=ListSets
