VAOMonitor Home Page

What's on the Page

1. Pending or active server notices. These are displayed at the top if any exist
2. The current status of every VAO service (science services, support services, and testing services). The status is either 'OK', or 'Fail' and in turn the status is also a link to the test history for a given service.
3. The current status of every Legacy service.
4. A list of any external (non-vao) centers that are currently failing. The information produced in this table comes from the External Monitor. To read about how this information is collected and displayed, please see the "External Sites" section below.
5. The current status of System services
6. The last time services were harvested from various registries
7. The status of Selenium tests run on VAO Science services
8. The date and time services were last monitored and the next scheduled monitor check.

Capabilities:

1.The VAO Science Monitor - a page that provides the status of key VAO services
http://heasarc.gsfc.nasa.gov/vo/vaomonitor/vaodb.pl?format=html

*This page is linked to from the usvao home page.

2.An xml representation of the status of all VAO services and System services.
*Note, the xml response does not include Legacy services. The url for this feature is:
http://heasarc.gsfc.nasa.gov/vo/vaomonitor/vaodb.pl?format=xml

External Sites

The External Monitor ("http://heasarc.gsfc.nasa.gov/vo/external_monitor/vodb.pl") provides a number of capabilities that are used in the vaomonitor home page.

When the vaomonitor home page loads this section, the following takes place:
1) The external monitor is invoked with format=xml.The response is an xml containing the status of every external site tested. It is important to note that this is the overall status of a site, not a service.

2)The xml response contains the service name,status and authority id of the center.
E.g.
<Service>
<name>SVO CAB<name/>
<status>OK</status>
<authid>ivo://svo.cab/</authid>
</Service>

3)The xml is parsed and the total number of centers up,the total down,and the center names are returned to the vaomonitor page.

4)The vaomonitor sw uses some JavaScript to display a bar showing the number of centers up relative to the total number of centers

5) Pending Feature: A bar representing the fraction of all services organized by type relative to the total number of services (app. 9,000). How to do this is tbd but the general expected setup is:

a)A new authid will be added to the Services table in the VAOMonitor.
b)A nightly script will run and perhaps by using MySQL will get the total number of unique services and the total number of services for a unique authid.This will only work if we introduce an authid column into the Validation DB's Services table


*For more information on the external monitor, please see the documentation that comes with that system.
