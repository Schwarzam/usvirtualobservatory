Retrieving data from the registry (Slurp)

The first step is to retrieve the registry information from the STScI registry. To do this we use the get_datatype.pl program which slurps services from the STScI registry by datatype.
The tool:

a)reads in the datatypes listed in the local file "downloadtypes".
b)uses the queries that "downloadtypes" contains to download each data type from the registry

You can see what this file contains here.

The program loops over the data in the above file,runs the queries and dumps the data into discreet files with the naming convention "{type}_res":

Each file above contains a concatenated list of resources. The resources in each file are described by the file name (E.g. con_res contains Cone resources). It is important to note that these files contains raw xml data. Another step which is handled by the parse_registry_resources.pl program is required to extract the metadata from each raw xml.

Manually running get_datatypes.pl

The data download from the registry can be run manually. The syntax for running this program on the command line is:

./get_datatypes.pl (this returns all recognized types defined in the downloadtypes file...dumping everything to distinct files).
./get_datatypes.pl -t {type} (enter a recognized type: cone, siap, etc.)
