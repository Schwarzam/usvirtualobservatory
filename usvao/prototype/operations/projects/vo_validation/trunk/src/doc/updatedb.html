This program uses the Cone, SIAP,and Registry processed files generated by parse_registry_resources.pl to update the database once per month. This program only adds new entries to the database and mails the user a list of identifiers that need to be removed from the registry. It has no other function.

The general logic behind this program's design is to:

A) read all of the services currently stored in the database into memory
B) read all of the services of a given type (from the download) into memory.
C) Compare A and B and add any new services to the Database (if possible)
D) Repeat B and C until we have compared everything downloaded from the registry to everything that we have stored in the db.

Mail the user a list of resources that should be deleted.
