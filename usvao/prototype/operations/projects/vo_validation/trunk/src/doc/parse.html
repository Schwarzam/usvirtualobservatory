Parse:

The primary function of this program is to parse out each unique record from the Cone, SIAP, and Registry XML files created by the get_datatype.pl program. The output to this program should be three unique files that contain metadata of the form:

"shortname|ivoid|serviceURL|type|RA|DEC|SR|role|fulltypename"

shortname- the shortname of the resource
ivoid- the ivo identifier for the record
serviceURL - the accessURL for the record
type - the service type (right now there are only 3 possibilities: cone,siap, registry)
RA,DEC - coordinates
SR - search radius
role - should be std (standard)
fulltypename -the full typename

*More information can be added to the above structure but for now it contains only the default elements that must be added to the database.

We can now use the above structure to update our database at the beginning of every month. The update is handled by the updatedb.pl script.

Manually running the parse_registry_resources.pl script.

If the files generated by get_datatype.pl have been created in the "dump" directory,then you can run parse_registry_resources.pl on any or all of them.

The syntax for runnning the script is:
./parse_registry.pl (runs on all recognized types)
./parse_registry.pl -t {type} ( runs on the type specified on the command line)

E.g. ./parse_registry_resources.pl -t cone

...generates file:

cone_res_for_validation
