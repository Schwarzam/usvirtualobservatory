The DALServer Toolkit

DALServer provides both reference code for the DAL services, enhancing the service specifications by illustrating how the service specifications can be implemented, as well as a toolkit which can be used to build actual services. DALServer deals only with the specifics of the DAL protocols; it is not a complete implementation, and does not dictate policy or assume anything about the host environment in which it is being used (in other words, it is intended to be integrated into a variety of different production environments). IVOA DAL services can be implemented with many different technologies or languages. The DALServer toolkit and reference implementation presented here supports Java-based services.

DALServer is still under development. The initial implementation supports only the simple spectral access protocol (SSAP), and was developed in parallel with the SSAP specification. The JHU proxy service has thus far only been tested for SDSS DR5. The JHU proxy service is intended primarily to provide access to real data for SSAP development, not for serious scientific use; JHU will provide a production data access service later for serious use.

About the Query Form

The query form is intended only to provide a simple means to test or get a quick look at the SSAP services (in particular we need to look at the details of the interface as shown here in order to finalize the SSAP specification). Client software which can construct more sophisticated queries and do interesting things with the data which is returned is needed for more serious use. It is possible however, to use the query form to get at data. To download data one can use the access reference URLs given in the query response (look for "acRef" in the XML output) to download the reference data.
