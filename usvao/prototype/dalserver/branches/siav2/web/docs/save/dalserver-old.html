The DALServer Toolkit

DALServer provides both reference code for the DAL services, augmenting the service specifications by illustrating how a service can be implemented, as well as a user toolkit which can be used to build actual services. DALServer deals mainly with the VO data access protocols used to query for and access data. While generic capabilities are provided for returning actual datasets, in general local extensions may be required to access, subset or transform the native data provided by a site. IVOA data services may be implemented with any technology or language. The DALServer toolkit and reference implementation presented here supports Java-based services.

DALServer is still under development. The current implementation supports access to tables (simple cone search or SCS), images (SIA), spectra (SSA), and spectral line lists (SLAP). The toolkit includes built-in test/demo services for each such class of data. Access to local data requires that the data provider configure additional service instances.

A precompiled snapshot (with sources) of the current DALServer toolkit distribution can be found on the VAO DALServer product page. If you do anything with this, please check for updates first, and report any problems encountered.

About the Query Form

The default query form provides a simple means to test or get a quick look at each class of service. Client software which can construct more sophisticated queries is needed for more serious use. It is possible however, to use the query form to get at data. To download data one can use the access reference URLs given in the query response (look for "acRef" in the XML output) to download the reference data.
