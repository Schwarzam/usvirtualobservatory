VAO Catalog Cross-Comparison Tool

Overview

This service performs spatial cross-comparisons between two tables, one supplied by the user and one an on-line source catalog, for a user-specified match radius. The service returns cross-identifications; that is, it returns all sources in the on-line catalog that are within the match radius of each input source. The output is a composite table consisting of records from the first table, joined to all the matching records in the second table, and the angular distance and position angles of the matches. Matches are not unique because there are often multiple sources in the cone of comparison, so the same source (from any of the catalogs) may appear in multiple places in the output, especially for crowded regions of the sky. The service returns a separate output table showing sources in the input that had no matches in the on-line catalog

This service is recommended for use with the most recent versions of Firefox, Chrome or Safari. Internet Explorer 9, old versions of Firefox (prior to version 8) and Opera are not supported.

Tutorial

The following is a step-by-step guide on the use of the VAO Cross-Comparison Tool. You may view video tutorials on how to get started and how to start analyzing your output with VAO tools by clicking the links below. If you need further assistance, you can contact the VAO Help desk.

 

Step 1. Launch the VAO Cross-Comparison Tool from the VAO Science Tools page.


Step 2. The Cross-Comparison Tool home page will appear.

Step 3. Upload a table of sources with the Upload a Table button. A dialog box will appear, allowing the user to select the desired file from the local machine.




Once selected, the file name appears next to the Upload button, along with a progress indicator and the option to cancel the upload. Note that the table must follow the formatting rules.



Step 4. Once the file has been uploaded, select it from the User Tables pane on the left by clicking on the file name. Next, select a catalog name from the On-Line Catalogs pane to the right. Only one on-line catalog may be selected at a time for comparison with the uploaded file.



Step 5. Enter a search radius (in arcsec) in the Max Match Distance field. The default value is 3 arcsec. To execute the cross-comparison, click the adjacent "Compare" button. A "processing, please wait" message appears on the screen while the comparison is being executed.



Step 6. Once completed, the upper-right field of the service display will be populated with the results of the comparison. It reports the number of records matched, the maximum, minimum, and average distances of the matches, file links for both matched objects and unmatched objects, as well as a file containing bad input data. The total processing time is also displayed.



Step 7. Select the download file format from the "Download Format" tab at the bottom of the field. The options are Aligned ASCII, CSV, tab-delimited or bar-delimited text, or VOTable format. Download by clicking the file link.



Step 8. The process may be repeated to compare the input table against other on-line catalogs in the list. Comparisons appear in the History panel at the bottom of the page. Uploaded data and comparison results remain available in subsequent sessions in that browser for up to four days.



Step 9. To view a previous comparison, click on the desired .tbl file in the History panel to select, then click the Restore Details tab. To remove an uploaded reference file and/or delete a comparison result file, simply click on the file's Delete checkbox.

SAMP: Simple Application Messaging Protocol

Note: This is a beta release of the SAMP functionality. Please let us know what you think.

SAMP is a mechanism that allows applications on your desktop (including web-based applications like the Cross-Comparison service) to share information, and to do so without necessarily knowing anything about each other. This is enabled by having a "third-party" application called a "SAMP hub" running locally on a well-known port. Any other application is then at liberty to register with this hub to get messages of a certain type (e.g. "I've got a table at this URL") or to send messages to the hub to be forwarded to any other application that has registered a desire to see them.

The first step is to activate a hub. Some VO applications (e.g. TOPCAT) automatically start a hub on launch. If not, you need to activate one yourself. There are a few different ways to do this but probably the simplest way is to activate the JAVA Webstart JSAMP hub at

   http://astrojs.github.com/sampjs/hub/webhub.jnlp

More information on this can be found here: http://astrojs.github.com/sampjs

Before the Cross-Comparison service can use SAMP, it must register with the hub. To do so, simply click the Register button in the middle of the page.




The only messages the Cross-Comparison service will be registered to receive are those that say a new table is available for download. Such messages can be triggered by various services (like the VAO Data Discovery tool). The Cross-Comparison service will treat such data like manually uploaded files and add their names to the "User Table."

Tables can also be broadcast from the Cross-Comparison service. The results summary panel now includes links for all the resulting tables that broadcast the URL of any result table (e.g. matches.tbl). Click this and (assuming you have some other tools listening) the path to the data file is sent to the external tool(s).
