HTTP Interface: Listing Uploaded Tables

There is a very simple interface to aid in keeping track of uploaded tables. The base URL for the listing service is:

	   http://vao-web.ipac.caltech.edu/cgi-bin/VAOSCC/nph-vaosccDir

It only has one parameter: "workspace". In fact, when working through a browser where this same parameter is carried in a cookie no parameters are needed at all.

The return structure from the listing service is simple XML (currently no schema):

      <results>
	 <status> OK </status>
	 <workspace> TMP_8qsgPp_9657/Upload </workspace>

	 <file>
	    <name> Messier.tbl </name>
	    <nrec> 14 </nrec>
	    <ncol> 15 </ncol>
	 </file>

	 <file>
	    <name> big.tbl </name>
	    <nrec> 476 </nrec>
	    <ncol> 15 </ncol>
	 </file>

	 <file>
	    <name> tenK.tbl </name>
	    <nrec> 99998 </nrec>
	    <ncol> 95 </ncol>
	 </file>

	 <file>
	    <name> xxx.tbl </name>
	    <nrec> 11 </nrec>
	    <ncol> 10 </ncol>
	 </file>

	 <file>
	    <name> abell.tbl </name>
	    <nrec> 2712 </nrec>
	    <ncol> 24 </ncol>
	 </file>

	 <nfile> 5 </nfile>

      </results>

The "workspace" is reflected here, for completeness. Then there is one parameter block per uploaded file, giving the "name" (a handle usually reflecting the file name on the original system but not the name as stored) plus sizing information. Finally there is a total file count.
