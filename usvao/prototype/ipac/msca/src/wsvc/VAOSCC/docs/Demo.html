VAO Cross-Comparison Tool Demo: Finding Brown Dwarfs

Overview

This service performs fast positional cross-matches between an input table of sources and common astronomical source catalogs. The example used here is to cross-compare a table of sources extracted from the WISE preliminary data release catalog with the SDSS DR9 database, in an effort to identify new brown/dwarf candidates.

The service does not return cross-identifications, but rather cross-identification candidates. That is, it returns a table of all SDSS DR9 sources within a user-specified radius of each WISE source. See the User Guide for details.

Usage

There are two ways of using the service: either through a simple web form, which will be used for this demonstration, or through a program interface that replicates all the functionality of the web form.

The service runs under the Firefox, Chrome and Safari browsers. We recommend avoiding Internet Explorer.

Demo

1. Bring up the VAOSCC service. Click the launch button next to Cross-Comparison Tool.

2. Upload the input table, which you can find and download to your desktop at: http://vao-web.ipac.caltech.edu/data/VAO/wise_subset.tbl. This sample input file contains 33,901 sources in a box 2° on a side centered at 15h 30m, +24°. The data were downloaded from the WISE Preliminary Data Release Catalog at IRSA using the Gator query engine.

  • The table is in column-delimited IPAC table format, but the service accepts input in many common table formats, such as CSV, tab-delimited, VOTable, and FITS tables.
  • The service will accept uploaded tables of up to 1 million records in size, and you can cross-match it with catalogs containing records of more than 1 billion records.
  • The service uploads the catalog to the server and validates it for consistency of format and content.

3. Highlight the input table, wise_prelim.wise_prelim_p3as_psd3011.tbl, under User Tables.

3. Highlight the SDSS DR 9 catalog in the list of On-line catalogs.

5. Select a "match-radius" of 10 arcsec. This large radius has been chosen because the WISE and SDSS surveys are separated by more than 10 years, and many brown dwarfs have appreciable proper motions.

6. Click Compare. The service reports that it is processing.

7. When done, the service returns three tables:

  • a. Matches.tbl: a table of all the SDSS sources within 10 arcsecs of each WISE source. The service conjoins the SDSS DR9 records with the WISE records so that no information in the two catalogs is lost.
  • b. Unmatched.tbl: records of WISE sources that have no SDSS DR 9 counterparts within the 10 arcsec match radius.
  • c. Bad.tbl: records of WISE sources that cound not be processed.

Note: This example processes in a few seconds. For large tables, typical times are:

  • a. 600,000 records: 500 s
  • b. 1,000,000 records: 825 s

8. Download the matches table. Select Aligned ASCII from the Download Format menu, then click the matches.tbl link to save the table to your local disk.

9. You can also see the job has been summarized in the History section at the bottom of the web page. This information will be stored for four days and then deleted from the server. As long as you have Web browser cookies enabled, you will be able to access these results again during that time. No one else will be able to see these records unless they have access to your Web browser.

Analysis and filtering of the file to identify candidate brown dwarfs is the astronomer’s responsibility, but the VO provides tools to help with this. The next video demonstrates one of them.

Analyzing the Cross-Match Table with TOPCAT

 

The VO provides tools to aid in the analysis of tabular data. One of the most powerful is the Tool for OPerations on Catalogues And Tables (TOPCAT), a desktop application available for free download. This demonstration shows how you may begin visualizing and filtering the cross-match output file with TOPCAT.

1. If you do not have TOPCAT installed, you may download it from http://www.star.bris.ac.uk/~mbt/topcat/. Installation instructions are at http://www.star.bris.ac.uk/~mbt/topcat/#install. For this demo, download the topcat-full.jar file and install it on your desktop. (There are other options for downloading.)

2. Launch TOPCAT by double-clicking on the topcat-full.jar file.

3. You can easily download the output table matches.tbl on to your desktop and the load it into TOPCAT by the File -> Load option in TOPCAT, but in this demo, we will input the table directly into TOPCAT through the Simple Access Messaging Protocol (SAMP). When TOPCAT launches, it automatically launches a SAMP "hub" that enables communication between it and the cross-match service (this hub is invisible to the end-user). (Other applications may not launch a SAMP hub, and you may have to activate one yourself. The simplest way is to activate the JAVA Webstart JSAMP hub at http://astrojs.github.com/sampjs/hub/webhub.jnlp. Installation instructions are here: http://astrojs.github.com/sampjs/.)

4. Click the Register button in the bottom-left corner of your browser to register the cross-match service with SAMP hub.

5. On the cross-match output page, click the Broadcast button next to the matches.tbl file to send the file into TOPCAT.

6. Visualize the structure and contents of the output file. Click Display Table Cell Data (the button at the top of the menu) and note the following:

  • The records from the user table are prepended with "user_"
  • Dist = radial distance of SDSS source from WISE source
  • Pang = Position angle of SDSS source from WISE source (degrees, E of N)
  • Nm = number of matches for each user input source.

7. Visualize a color-color plot derived from the output table. Click Scatter Plot (the button) to bring up the plot window. By default, TOPCAT will display the RA and Declination of sources in the WISE table.

  • Under X axis, type user_w1mpro – user_w2mpro
  • Under Y axis, type SDSS_DR9_i – SDSS_DR9_z

The plot window now shows a color-color diagram of WISE [3.4 µm] – [4.6 µm] and SDSS i-z.

8. Brown dwarfs have colors of [3.4 µm] – [4.6 µm] >0.4 and i-z >2. Visualize this region of color-color space by adjusting the range of the axes. Click axes, got Range and enter:

  • X minimum = 0.3
  • X maximum = 2
  • Y minimum = 2
  • Y maximum = 10

Constrain the plots further using the X subrange and Y subrange sliders.
