This template illustrates all the capabilities of the "htmlgen" service. All substitution rules are based on variations on a \param\ pattern matching. The most basic capability is just a string substitution of the value associated with the param name on the htmlgen command line: example1.

The htmlgen command line starts off with two parameters defining the location of the template, e.g. this file test.template, and the output, which is usually HTML (see test.tbl). This is followed by parameter/value pairs. If the number of parameters is going to be large, the parameter/values can be written into a file and passed to the program with a param/value pair that looks like "@parms param.txt". All the pairs in this file are added to the param/value list as if they had been put on the command line.

Default values are also supported: \parm3:like this\ -> like this

You can also use config variables such as etss_corot_exo_v1_3 or /workspace/beekley_9135 where the value is retrieved from the ISIS.conf configuration file. If the parameter name starts with an ampersand (\&parm2\), the value is URL-encoded: e%2Bample2.

If the parameter name starts with a pound sign (\#servheader.html\) as at the top of this template, the text of the associated file is inserted into the output. The '#' indicates that the file is to be found in the standard include directory (set up for Apache and pointed to by the ISIS.conf HTML_INCLUDES variable).

Similarly, any file (full path) can be included using \@incfile\:

This is the text from file "include1.html".

There are also a few functions that can be applied to the named parameter:

sign1 = 1 < X
sign3 = -1 > Z

The htmlgen processing has special rules for handling tables. Here a table file is translated into an HTML table, but what the code actually does is loop over the table records and output one copy of the pattern between the \%tblfile\ and the \%%\, or the \%alt\ default if it exists. Inside this block, the \param\ values refer to table column names rather than command-line parameters, though the latter can also be accessed using a \p.param\ construct, as can ISIS.conf variables through \c.param\.
